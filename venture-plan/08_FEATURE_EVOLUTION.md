# 08 · המשך והצעות פרואקטיביות (גרסת שרת · ללא חשבונות)

> 3 הצעות ממוקדות — מה חסר, מה יחזק, ומה הצעד הבא. תואם לכיוון: אתר מבוסס‑שרת, חינמי, בלי משתמשים.

### 1. Programmatic SEO על דפי הרכב ה‑SSR — *מה יחזק*
עכשיו שיש SSR ו‑URL לכל רכב/דגם, לייצר אוטומטית דפי longtail ("ריקולים למאזדה 3", "צריכת דלק קורולה 2019")
מהנתונים. זה מנצל בדיוק את היתרון שהשרת נותן (רינדור אינדקסבילי) בלי שום עלות נוספת.
**רציונל:** ה‑SSR הוא ההשקעה; ה‑programmatic SEO הוא ה‑ROI שלה. **ערך גבוה / מאמץ בינוני.**

### 2. שער נגישות אוטומטי ב‑CI — *מה חסר (ולהשלים מיד)*
לחווט `@axe-core/playwright` + Lighthouse CI כך שכל PR נכשל אם הנגישות נשברת. הופך "נגיש" מהצהרה לערובה
מתמשכת + תג ב‑README.
**רציונל:** נגישות חובה חוקית; אוטומציה מונעת רגרסיה. גם מרשים בתיק עבודות. **ערך גבוה / מאמץ נמוך.**

### 3. תזכורת מייל ללא חשבון (S4) — *הצעד הבא הכי טוב, אם רוצים retention*
מנוי חד‑פעמי לרכב: מייל → אישור (double opt‑in) → תזכורת. מוסיף וו‑חזרה אמיתי בלי מלוא מערכת המשתמשים —
מבודד כך שהליבה נשארת חסרת‑DB.
**רציונל:** מגשר על הפער היחיד שנוצר מהסרת החשבונות (retention) בעלות מינימלית. **ערך בינוני‑גבוה / מאמץ בינוני.**

## פערים שנשמרו במודע
- אין חשבונות/DB/מודל מסחרי — לפי בחירתך. מסמכי הצד המסחרי (finance/metrics/GTM שיווקי) לא רלוונטיים.
- ריקולים ומספר בעלים — ממתינים לאימות מזהי מאגר; מוצג "לא זמין".

## מה מוכן עכשיו
תוכנית עבודה מלאה לגרסת שרת ללא חשבונות: פרופיל → אפיון → build spec (M0–M5) → אבטחה → DevOps → handoff
kit → מרשם סיכונים/הנחות. אפשר למסור את `05_BUILD_SPEC.md` (דרך `29_HANDOFF_KIT.md`) ל‑Claude Code ולהתחיל.
