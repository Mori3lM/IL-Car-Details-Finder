# 00 · פרופיל המוצר (מקור אמת משותף)

> קובץ הפרופיל שכל שאר המסמכים קוראים במקום לגזור הקשר מחדש.
> **כיוון סופי (יולי 2026):** אתר **מבוסס‑שרת, ללא מערכת משתמשים**. השרת עושה עבודה אמיתית — SSR ל‑SEO,
> תיווך (proxy) ל‑data.gov.il (פותר CORS), cache ובקרת קצב — אבל **אין חשבונות, אין login, אין DB של
> משתמשים**. שמירת רכבים נשארת מקומית (localStorage); תזכורת רישוי/טסט דרך קובץ `.ics`. הפרוטוטייפ
> ב‑`src/` הוא reference; תוכנית העבודה (`05_BUILD_SPEC.md`) נבנית מחדש לגרסת השרת.

## מהות המוצר
אתר ווב מבוסס‑שרת (Next.js SSR) לבדיקת רכב ישראלי לפי מספר רישוי. ה‑backend מתווך אל data.gov.il, שומר
תוצאות ב‑cache, ומגיש דפי רכב מרונדרים בשרת (טובים ל‑SEO ולשיתוף). **בלי הרשמה** — ערך מיידי לכל מבקר.

| ממד | ערך |
|---|---|
| סוג מוצר | אתר ווב מבוסס‑שרת (SSR + API routes), **ללא חשבונות** |
| פלטפורמה | ווב (מובייל‑פירסט), Next.js |
| שוק | ישראל, עברית, RTL |
| קהל | בעלי רכב יומיומיים (+ קוני יד שנייה) |
| מודל | לא‑מסחרי / פרויקט אישי רציני; **חינמי לחלוטין** (שכבת‑חינם, ללא DB) |
| רגישות נתונים | **נמוכה** — אין חשבונות ואין PII בשרת; cache אנונימי בלבד |
| נגישות | WCAG 2.1 AA — חובה חוקית (ת"י 5568) |
| ארכיטקטורה | Next.js (App Router) · route handler ל‑gov proxy · cache (Next revalidate / Upstash KV חינמי) · localStorage · `.ics` |

## למה שרת אם אין משתמשים?
השרת מצדיק את עצמו גם בלי חשבונות: (1) **פותר CORS** — הדפדפן לא קורא ישירות ל‑gov; (2) **SSR ל‑SEO** —
דפי רכב מרונדרים בשרת, אינדקסבילים ושיתופיים; (3) **cache ובקרת קצב** — מהירות והגנה על מכסת ה‑gov API;
(4) מקום עתידי להרחבות. זה בדיוק "אתר שרץ על שרת" — בלי המורכבות של מערכת משתמשים.

## Overlays / ארכיטיפים שחלים
- **Consumer web app** (SEO קריטי, מובייל‑פירסט, נגישות).
- **Third‑party data dependency** — תלות ב‑data.gov.il (סיכון פלטפורמה — ר' `27`).
- לא חל: חשבונות/Auth, DB יחסי, תשלומים, LLM/AI, enterprise, marketplace, health, web3.

## רמת מורכבות (Complexity Tier)
**T1.5 — קל‑בינוני.** יש backend (proxy+cache+SSR) ותלות חיצונית, אבל **אין auth, אין DB, אין jobs**.
עומק: אבטחת שרת ממוקדת (validation, no‑SSRF, rate‑limit, secrets), נגישות מלאה, DevOps קל. בלי מודל
פיננסי/validation מסחרי (לא‑מסחרי).

## הכרעות מפתח (הפיכות מסומנת)
- **Next.js SSR (בלי DB/auth)** — *הפיך יחסית*. framework אחד = שרת + לקוח + SSR ל‑SEO.
- **Cache: Next revalidate / Upstash KV חינמי** — *הפיך*. בלי Postgres בליבה.
- **שמירה ב‑localStorage + תזכורת `.ics`** — *הפיך*, אפס PII, אפס עלות.

> ASSUMPTION: אין חשבונות ⇒ אין מיילים ⇒ **תזכורת = הורדת `.ics`** (היומן של המשתמש שולח את ההתראה).
> אם בעתיד תרצה "תזכורת במייל בלי חשבון" (מנוי חד‑פעמי לרכב: מייל→אישור→תזכורת) — זו תוספת אופציונלית
> שתדרוש DB קטן + Resend + cron. סומן כ‑Should אופציונלי, כבוי כברירת מחדל.

> ASSUMPTION: פרויקט **לא‑מסחרי**, חינמי לחלוטין, ברמת production.
