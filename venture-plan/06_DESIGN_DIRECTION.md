# 06 · כיוון עיצוב וחוויית משתמש

> "make it beautiful" — הבידול העיצובי הוא נכס אמיתי בשוק שנראה מיושן. מובייל‑פירסט, RTL מוקפד, נקי.

---

## 1. עקרונות עיצוב

1. **בהירות לפני עושר.** מספר רישוי נכנס, תשובה קריאה יוצאת. אפס עומס.
2. **אמון חזותי.** מקורות רשמיים מסומנים, סטטוסים בצבע ברור (טסט בתוקף/פג, ריקול פתוח/אין). המשתמש צריך להרגיש "זה אמין".
3. **מובייל‑פירסט אמיתי.** רוב החיפושים בנייד, לרוב תוך כדי עמידה מול רכב. כפתורים גדולים, קלט מספרי, תוצאה בגלילה אחת.
4. **RTL כאזרח מדרגה ראשונה.** לא "אתר אנגלי שהופך" — שימוש ב‑CSS logical properties, שיקוף אייקונים כשצריך, מספרים/תאריכים בפורמט ישראלי.

## 2. רגע ה‑"aha"

הרגע שבו המשתמש "מבין": מיד אחרי החיפוש הראשון, כרטיס הרכב מופיע יפה ומאורגן — **ומתחתיו כרטיס סטטוס אחד גדול: "הטסט שלך בתוקף עד 14.11.2026 · עוד 132 יום"** עם CTA יחיד ובולט: **"שמור את הרכב וקבל תזכורת בזמן"**. שם הופכים מבקר חד‑פעמי לבעלים רשום. זה הציר של כל המוצר.

## 3. מערכת עיצוב (Design Tokens)

צבע — נקי ואמין, מבטא "רשמי אך ידידותי":

```
--brand-600:  #1E6FB8   /* כחול אמון — פעולות ראשיות, קישורים */
--brand-500:  #2E86D3
--brand-050:  #EAF2FB   /* רקע כרטיס פעיל */
--ink-900:    #0F172A   /* טקסט ראשי */
--ink-500:    #64748B   /* טקסט משני */
--surface:    #FFFFFF
--bg:         #F6F8FB   /* רקע עמוד */
--ok-600:     #178A5A   /* תקף / אין ריקול */
--warn-600:   #B7791F   /* מתקרב לפקיעה */
--danger-600: #C2410C   /* פג / ריקול פתוח */
--border:     #E2E8F0   /* גבול דקורטיבי לכרטיסים (הפרדה ויזואלית בלבד) */
--border-ui:  #8593A6   /* גבול לרכיבים אינטראקטיביים (שדות קלט) — עומד ב‑3:1 לפי WCAG 1.4.11 */
--radius:     14px       /* פינות רכות, מודרני */
```

טיפוגרפיה: `Heebo` או `Assistant` (גופני עברית פתוחים, קריאים, מודרניים) מ‑Google Fonts. כותרת H1 ~28px/600, גוף 16px/400, line-height 1.6. מספרים בפונט tabular.

מרווח: סקאלת 4px (4/8/12/16/24/32). כרטיסים עם צל עדין (`0 1px 3px rgba(15,23,42,.08)`), גבול הפיקסל 1px.

אייקונוגרפיה: סט קווי אחיד (Lucide/Tabler), 20–24px, `currentColor`.

## 4. פריסת מסכי מפתח

- **דף בית:** כותרת ערך + שדה חיפוש ענק ממורכז + שורת אמון ("ממקורות משרד התחבורה · חינם"). מתחת: 3 כרטיסי הסבר קצרים (מה מקבלים).
- **עמוד תוצאה:** כרטיס זהות רכב (יצרן/דגם/שנתון + צבע) → כרטיס סטטוס טסט/רישוי (הכי בולט) → בעלים → ריקולים → יחידת ליד ביטוח → מקורות. CTA "שמור רכב" דביק (sticky) בנייד.
- **הרכבים שלי:** רשימת כרטיסים, כל אחד עם תג סטטוס צבעוני והתראות פתוחות.

## 5. מצבים (States) — לא לשכוח

טעינה (skeleton), ריק ("עדיין לא שמרת רכב"), שגיאה (מכובדת + ניסיון חוזר), נתון חסר ("לא זמין" מפורש), הצלחה. כל אחד מעוצב, לא ברירת מחדל של הדפדפן.

---

## 6. פריסה רספונסיבית ודסקטופ (תיקון מפידבק — האתר נראה קטן מדי בדסקטופ)

**הבעיה:** התוכן נעול ל‑`max-width` צר (~640–680px) וממורכז — טוב לנייד, אבל במסך מחשב זה נראה קטן, ריק
בצדדים, ולא "ממלא". צריך פריסה שמתרחבת עם המסך.

**העקרונות:**
- **קונטיינר גמיש:** `width: min(100% - 2rem, var(--container))`. `--container` גדל לפי breakpoint:
  ~680px בנייד/טאבלט קריאה → **~1100–1180px בדסקטופ**. תמיד עם padding צדדי.
- **טיפוגרפיה נוזלית (`clamp`)**: כותרות וגוף גדלים במעט בדסקטופ. דוגמה: H1 `clamp(1.6rem, 1.1rem + 2.2vw, 2.6rem)`, גוף `clamp(1rem, .96rem + .3vw, 1.15rem)`. כך הטקסט לא ננוׂס במסך גדול.
- **פריסת עמוד תוצאה בדסקטופ = שתי עמודות (≥ 900px):** עמודה ראשית רחבה (כרטיס זהות + פרטים טכניים
  בגריד 3–4 עמודות) לצד **עמודה צדדית דביקה (`sticky`)** עם כרטיס הסטטוס והפעולות (שמור / תזכורת). בנייד —
  עמודה אחת, סטטוס ופעולות ראשונים.
- **גריד פרטים מסתגל:** `grid-template-columns: repeat(auto-fit, minmax(180px, 1fr))` — 2 עמודות בנייד,
  3–4 בדסקטופ. מנצל רוחב.
- **דף בית בדסקטופ:** hero רחב עם כותרת גדולה + שדה חיפוש מרכזי גדול; מתחת, סקשנים ברוחב מלא (ר' §8).
- **יעדי מגע ≥44px** נשמרים; זום 200% נשמר; container queries מותרים היכן שמתאים.

## 7. סדר וקנה‑מידה של קטעי ה‑UI (לוודא סדר — מפידבק)

היררכיה עקבית בכל עמוד: **Header (עם ניווט + כפתור נגישות) → תוכן ראשי → Footer עשיר**. בעמוד התוצאה,
סדר קבוע: זהות הרכב → **סטטוס רישוי/טסט (הכי בולט)** → פרטים טכניים → בעלות → ריקולים → מקורות → פעולות.
רווחים אנכיים נדיבים בין קטעים (`clamp(1.5rem, 4vw, 3rem)`), כותרת‑קטע ברורה לכל בלוק, יישור אחיד ל‑RTL.

## 8. תוכן עשיר יותר (שהאתר ירגיש מלא ואמין — מפידבק)

להוסיף קטעי תוכן אמיתיים (גם מחזק SEO ואמון):
- **Hero** בדף הבית: כותרת ערך ("כל מה שהמדינה יודעת על הרכב — במספר אחד") + תת‑כותרת קצרה + שדה חיפוש + שורת אמון ("ממקורות רשמיים · חינם · ללא הרשמה").
- **"איך זה עובד"** — 3 צעדים (מזינים מספר → מקבלים כרטיס → שומרים ומקבלים תזכורת), עם אייקונים.
- **"מה מקבלים"** — כרטיסי פיצ'ר קצרים (פרטי רכב, סטטוס רישוי/טסט, שמירה, תזכורת ליומן).
- **שאלות נפוצות (FAQ)** — "האם זה חינם?", "מאיפה המידע?", "האם רשמי?", "מה לא זמין?", "האם המידע נשמר?"
  (שקיפות + מילות מפתח longtail). מומלץ `FAQPage` schema.org.
- **Footer עשיר** — קישורים: מאיפה הנתונים, פרטיות, **הצהרת נגישות**, וקרדיט ל‑data.gov.il + הבהרה.
- כל הטקסטים בעברית טבעית, ידידותית, לא שיווקית‑יתר.
