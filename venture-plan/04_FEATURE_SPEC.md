# 04 · אפיון פיצ'רים ו‑MVP (גרסת שרת · ללא חשבונות)

> **מחליף גרסאות קודמות.** אתר מבוסס‑שרת **בלי מערכת משתמשים**. השרת: SSR + proxy ל‑gov + cache + בקרת
> קצב. שמירה = localStorage (לקוח). תזכורת = `.ics`. אין auth, אין DB של משתמשים, אין PII בשרת.

---

## 1. סקופ — MoSCoW

**Must:**
- M1 · חיפוש רכב לפי מספר רישוי דרך **API של השרת** (`/api/vehicle/[plate]`, proxy ל‑data.gov.il).
- M2 · **cache בשרת** לתוצאות (Next revalidate / Upstash KV חינמי) + בקרת קצב לכל IP.
- M3 · **דף רכב מרונדר בשרת (SSR)** `‎/car/[plate]` — אינדקסבילי, שיתופי, מהיר; כרטיס רכב מעוצב.
- M4 · **שמירת רכבים מקומית** (localStorage) — "הרכבים שלי", בלי הרשמה, פרטי לחלוטין.
- M5 · **תזכורת `.ics`** לטסט/רישוי (היומן של המשתמש שולח את ההתראה) — אפס PII, אפס שרת‑תזכורות.
- M6 · נגישות **WCAG 2.1 AA** מאומתת (ר' `12`), RTL, מובייל‑פירסט.
- M7 · SEO: `schema.org/Vehicle`, meta, sitemap, robots על דפי הרכב ה‑SSR.
- M8 · שקיפות מקור + עמוד "מאיפה הנתונים" + עמוד פרטיות + הבהרה.

**Should:**
- S1 · חיפוש לפי VIN/שלדה.
- S2 · ייצוא/הדפסת כרטיס ל‑PDF (דרך הדפדפן).
- S3 · מצב כהה נגיש.
- S4 · *(אופציונלי, כבוי כברירת מחדל)* תזכורת במייל **בלי חשבון** — מנוי חד‑פעמי לרכב (מייל→אישור→תזכורת).
  דורש DB קטן + Resend + cron. רק אם תבחר להפעיל.

**Could:**
- C1 · אינטגרציית ריקולים + מספר בעלים (כשמזהי המאגר יאומתו).
- C2 · השוואת שני רכבים.

**Won't (בגרסה זו):**
- חשבונות/login, DB של משתמשים, תשלומים/מנויים, אפליקציה נייטיב, ריבוי שפות, פאנל אדמין.

## 2. סיפורי משתמש (Must) + קריטריוני קבלה

### US‑1 · חיפוש דרך השרת
**קבלה:** הלקוח קורא ל‑`/api/vehicle/[plate]` (לא ישירות ל‑gov); השרת מנרמל, מאמת (Zod), מושך מ‑gov,
ממפה לעברית, מחזיר JSON טיפוסי; לא קיים → 404 ידידותי; כשל gov → 502 מטופל (בלי stack); זמן < 1ש' עם cache.

### US‑2 · Cache ובקרת קצב (שרת)
**קבלה:** תוצאה נשמרת ב‑cache עם TTL (למשל 24ש') ומוגשת ממנו בתוך ה‑TTL בלי לקרוא ל‑gov; מוצג
`source_updated_at`; בקרת קצב לכל IP מונעת enumeration והגנה על מכסת gov.

### US‑3 · דף רכב SSR (SEO)
**קבלה:** `‎/car/[plate]` מרונדר בשרת — `view-source` מציג את הנתונים; `schema.org/Vehicle`, meta ו‑OG
תקינים; הדף שיתופי (URL לכל רכב).

### US‑4 · שמירת רכב (localStorage)
**קבלה:** "שמור רכב" מוסיף ל"הרכבים שלי" ב‑localStorage; נשמר בין ביקורים באותו דפדפן; אפשר להסיר/לנקות;
הודעה שהנתונים מקומיים בלבד; שום דבר לא נשלח לשרת.

### US‑5 · תזכורת `.ics`
**קבלה:** מחושב תאריך יעד מתוקף הרישוי; "הוסף תזכורת ליומן" מוריד `.ics` תקין עם התראה N ימים לפני
(ברירת מחדל 30); נפתח ב‑Google/Apple/Outlook; אפס PII, אפס backend.

### US‑6 · נגישות (חובה — מאומתת)
**קבלה:** ר' `12`. בקצרה: ניגודיות ≥4.5:1, מקלדת מלאה + focus גלוי, סטטוס בטקסט+אייקון (לא רק צבע),
סמנטיקה/ARIA, קורא‑מסך, 320px, `lang=he dir=rtl`. axe נקי + Lighthouse a11y ≥95.

### US‑7 · שקיפות ופרטיות
**קבלה:** מקור + תאריך לכל בלוק; עמוד "מאיפה הנתונים" (כולל מה לא זמין); עמוד פרטיות (cache אנונימי,
localStorage מקומי, אין PII); הבהרה שהמידע אינפורמטיבי ואינו תחליף לבדיקה פיזית.

## 4. זרימות
1. **חיפוש:** דף בית → מספר → קריאה לשרת → כרטיס (SSR ב‑`/car/[plate]`) → "שמור" / "תזכורת ליומן".
2. **חוזר:** "הרכבים שלי" (localStorage) → סטטוס רישוי/טסט.
3. **קונה:** חיפוש → כרטיס → דגלים → הדפסה/PDF/שיתוף URL.
